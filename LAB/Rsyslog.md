# Лабораторная работа № Y
## Централизованный сбор и анализ системных журналов

**Специальность:** Сетевое и системное администрирование  
**Курс:** 3  
**Операционная система:** Debian 12  
**Уровень сложности:** средний  
**Форма выполнения:** индивидуально  

---

# 1. Цель лабораторной работы

Получить практические навыки настройки централизованного сбора системных журналов (логов) с нескольких серверов Linux и анализа событий инфраструктуры.

---

# 2. Задачи лабораторной работы

В процессе выполнения студент должен:

- развернуть сервер централизованного логирования;
- настроить rsyslog для приема удалённых журналов;
- настроить отправку логов с клиентских серверов;
- организовать хранение логов по отдельным каталогам;
- проверить передачу системных сообщений;
- выполнить анализ полученных логов.

---

# 3. Теоретические сведения

## 3.1 Что такое системные журналы

Системные журналы (logs) — это записи о событиях операционной системы и приложений.

Они позволяют:

- диагностировать ошибки;
- отслеживать действия пользователей;
- обнаруживать попытки несанкционированного доступа;
- анализировать работу сервисов.

---

## 3.2 Централизованное логирование

Централизованное логирование — это процесс отправки журналов с нескольких серверов на один центральный сервер.

Преимущества:

- удобство анализа;
- упрощение администрирования;
- повышение безопасности;
- сохранность журналов при сбое сервера.

---

## 3.3 Система rsyslog

`rsyslog` — стандартная система журналирования в Linux.

Функции:

- сбор системных журналов;
- передача логов по сети;
- фильтрация сообщений;
- хранение логов.

---

# 4. Схема лабораторной установки

Используются 4 виртуальные машины:

```

+----------------------+
| Log Server           |
| logs.studentXX       |
| 192.168.XX.50        |
+----------+-----------+
|
-

|                 |               |
+-------+-------+ +-------+-------+ +-------+-------+
| DNS Server     | | Mail Server    | | Zabbix Server |
| 192.168.XX.10   | | 192.168.XX.20  | | 192.168.XX.30 |
+----------------+ +----------------+ +----------------+

```

Все серверы отправляют системные журналы на **Log Server**.

---

# 5. Исходные данные

| Параметр | Значение |
|----------|----------|
| Сервер логирования | logs.studentXX |
| IP адрес | 192.168.XX.50 |
| Протокол передачи | UDP 514 |
| Каталог хранения логов | /var/log/remote |

---

# 6. Ход работы

---

# Шаг 1. Подготовка Log Server

Установить Debian 12.

Задать имя хоста:

```bash
hostnamectl set-hostname logs.studentXX.local
```

Обновить систему:

```bash
apt update && apt upgrade -y
```

Проверить наличие rsyslog:

```bash
systemctl status rsyslog
```

Если служба отсутствует:

```bash
apt install rsyslog -y
```

---

# Шаг 2. Настройка сервера приема логов

Открыть конфигурационный файл:

```bash
sudo nano /etc/rsyslog.conf
```

Найти и раскомментировать строки:

```bash
module(load="imudp")
input(type="imudp" port="514")
```

Это включает прием логов по UDP.

---

# Шаг 3. Настройка каталога хранения логов

Создать каталог:

```bash
sudo mkdir /var/log/remote
```

Добавить правило в конец файла `/etc/rsyslog.conf`:

```
$template RemoteLogs,"/var/log/remote/%HOSTNAME%/%PROGRAMNAME%.log"
*.* ?RemoteLogs
& stop
```

Теперь логи будут храниться по отдельным каталогам для каждого сервера.

---

# Шаг 4. Перезапуск службы

```bash
sudo systemctl restart rsyslog
```

Проверить статус:

```bash
systemctl status rsyslog
```

---

# Шаг 5. Настройка клиента (DNS Server)

Открыть файл:

```bash
sudo nano /etc/rsyslog.conf
```

Добавить строку:

```
*.* @192.168.XX.50:514
```

Это означает отправку всех журналов на Log Server.

---

# Шаг 6. Перезапуск службы

```bash
sudo systemctl restart rsyslog
```

---

# Шаг 7. Настройка остальных серверов

Повторить те же действия на:

* Mail Server
* Zabbix Server

Добавив строку:

```
*.* @192.168.XX.50:514
```

---

# Шаг 8. Проверка передачи логов

На клиентском сервере выполнить:

```bash
logger "Test log message from DNS server"
```

---

# Шаг 9. Проверка на Log Server

Перейти в каталог логов:

```bash
cd /var/log/remote
```

Просмотреть файлы:

```bash
ls
```

Проверить содержимое:

```bash
cat /var/log/remote/DNSSERVER/syslog.log
```

или

```bash
tail -f /var/log/remote/*/*.log
```

---

# Шаг 10. Анализ логов

Найти ошибки:

```bash
grep error /var/log/remote/*/*.log
```

Найти SSH-подключения:

```bash
grep sshd /var/log/remote/*/*.log
```
